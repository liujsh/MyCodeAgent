# MyCodeAgent 项目概览

## 项目定位与目标

MyCodeAgent 是一个**面向学习与实验的代码代理框架**，聚焦于**工具协议**、**上下文工程**、**子代理机制**与**可观测性**的系统化实践。

**核心目标**：让"Agent 能做什么"与"Agent 为什么能做"都可追溯、可验证、可扩展。

**适用场景**（基于 `README.md`）：
- 学习 function calling + 工具协议的真实落地
- 研究上下文工程（截断、压缩、持久化）
- 实验 Skills / Task 子代理协作
- 快速搭建可扩展的本地 Agent 试验场

## 主要功能特性

### 1. Function Calling 工具调用

**特点**：
- 不依赖 Action 文本解析，完全基于 OpenAI function calling 规范
- 支持工具调用的结构化参数传递与返回值解析
- 工具调用与响应通过 `tool_calls` / `tool` 角色消息配对

**参考文件**：
- `agents/codeAgent.py` - ReAct 循环实现
- `core/llm.py` - LLM 调用与 function calling 解析

---

### 2. 统一工具响应协议

**协议规范**（`docs/通用工具响应协议.md`）：

所有工具返回必须遵守以下结构：

```typescript
interface ToolResponse {
  status: "success" | "partial" | "error";
  data: Record<string, any>;
  text: string;
  error?: { code: string; message: string };
  stats: { time_ms: number; [key: string]: number | string };
  context: { cwd: string; params_input: any; path_resolved?: string };
}
```

**状态语义**：
- `success`：任务完全按预期执行
- `partial`：结果可用但存在折扣（截断、fallback、dry-run、部分失败）
- `error`：工具无法提供有效结果（访问拒绝、参数错误、超时等）

**标准错误码**：
- `NOT_FOUND`, `ACCESS_DENIED`, `PERMISSION_DENIED`, `INVALID_PARAM`
- `TIMEOUT`, `INTERNAL_ERROR`, `EXECUTION_ERROR`, `CONFLICT`
- `IS_DIRECTORY`, `BINARY_FILE`, `CIRCUIT_OPEN`
- MCP 系列：`MCP_PARAM_ERROR`, `MCP_PARSE_ERROR`, `MCP_EXECUTION_ERROR`, `MCP_NETWORK_ERROR`, `MCP_TIMEOUT`, `MCP_NOT_FOUND`

**参考文件**：
- `docs/通用工具响应协议.md` - 完整协议规范
- `tools/base.py` - 工具基类实现

### 4. Task 子代理机制（MVP）

**设计目标**（`docs/task(subagent)设计文档.md`）：
提供最小子代理系统，使主代理可以将专注工作委托给子代理。

**子代理类型**（4 种）：
1. **general** - 通用子代理
2. **explore** - 代码库扫描与入口点定位
3. **plan** - 实现步骤生成与依赖分析
4. **summary** - 长输出压缩与多文件发现总结

**模型选择**：
- `main` - 用于复杂推理或计划密集型任务
- `light` - 用于简单任务（总结、快速查找）

**工具权限隔离**：
- **禁用列表**：Task, Write, Edit, MultiEdit, Bash
- **允许列表**：LS, TodoWrite, Glob, Grep, Read

**执行流程**：
1. 主代理决定委托工作
2. Task 工具验证输入
3. 根据 `subagent_type` 选择角色提示词
4. 构建子代理工具集（应用 deny list）
5. 使用指定模型（main 或 light）运行子代理
6. 返回结果 + 工具摘要

**参考文件**：
- `docs/task(subagent)设计文档.md` - 完整设计文档
- `tools/builtin/task.py` - Task 工具实现
- `prompts/agents_prompts/subagent_*.py` - 子代理提示词

### 5. AgentTeams 团队协作（实验性）

**设计目标**（`docs/agent_teams/AgentTeams功能设计文档.md`）：
在 `CodeAgent + ToolRegistry + TaskTool` 架构上提供可用的会话团队能力。

**核心功能**：
- 团队创建、消息通信、并行分工执行
- 状态观测和清理
- 支持共享任务板（依赖管理、claim 机制）
- 消息 ACK 三态：pending / delivered / processed
- work item 状态：queued / running / succeeded / failed / canceled

**Task 三种模式**：
1. `mode=oneshot` - 一次性子代理（默认，兼容旧行为）
2. `mode=persistent` - 创建持久 teammate（参数：team_name + teammate_name）
3. `mode=parallel` - 快捷并行分发（参数：team_name + tasks）

**存储模型**：
- `.teams/<team>/config.json` - 团队配置
- `.teams/<team>/<member>_inbox.jsonl` - 消息存储
- `.teams/<team>/work_items/work_items_<teammate>.jsonl` - 并行工作项
- `.tasks/<team>/task_<id>.json` - 共享任务板

**Feature Flag**：
- `ENABLE_AGENT_TEAMS=true` 启用（默认关闭，便于快速回滚）
- `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 兼容开关

**参考文件**：
- `docs/agent_teams/AgentTeams功能设计文档.md` - 完整设计文档

### 6. 上下文工程

**分层结构**（`docs/上下文工程设计文档.md`）：

```
L1: 系统静态层
  └─ System Prompt + 工具提示词（固定前缀，稳定）

L2: 项目规则层
  └─ CODE_LAW.md（有则注入，无则为空；不进入 history）

L3: 会话历史层
  └─ user / assistant / tool 消息 + system summary

+ 当前 user 输入（本轮）
+ Todo recap（若有）
```

**核心特性**：

#### 6.1 历史压缩（History Compaction）
- **压缩目标**：只压缩 user / assistant / tool 消息
- **保留规则**：至少保留最近 10 轮（可配置），必须对齐完整轮次
- **工具调用完整性**：tool_use / tool_result 必须成对保留
- **Summary 生成**：旧历史归档为 system message，不会被再次压缩

#### 6.2 工具输出截断
- **统一策略**（`docs/工具输出截断设计文档.md`）：
  - `MAX_LINES = 2000`
  - `MAX_BYTES = 50KB`
  - 方向：`head`（默认）或 `tail`
- **处理流程**：
  1. 行数 ≤ MAX_LINES 且字节数 ≤ MAX_BYTES → 原样返回
  2. 否则 → 截断 + 落盘保存完整输出 + 返回提示路径
- **框架级截断**：ObservationTruncator 在写入 history 前统一截断

#### 6.3 @file 处理（Kode 风格）
- **原则**：@file 只触发提醒，不注入文件内容；必须先 Read
- **流程**：
  1. 解析 `@path`
  2. 注入 system-reminder（不含文件内容）
  3. 模型调用 Read 获取内容
  4. Read 结果进入 history（含 mtime）
- **system-reminder 模板**：
  ```
  <system-reminder>
  The user mentioned @src/utils/auth.ts.
  You MUST read this file with the Read tool before answering.
  </system-reminder>
  ```

#### 6.4 文件新鲜度追踪
- Read 返回 mtime
- 再次读取同一文件且 mtime 变化时追加提示：`Note: <file> was modified externally.`

#### 6.5 Token 触发规则
- **Token 来源**：上一次 API 调用的 `usage`（精确）
- **前置估算**：当前 user 输入 `len(text)//3`
- **Context Window**：默认 128,000（可通过环境变量调整）
- **阈值**：`0.8 × context_window`
- **最低消息数**：至少 3 条消息才考虑压缩

- `docs/上下文工程设计文档.md` - 完整设计文档
- `docs/工具输出截断设计文档.md` - 截断策略细节

### 7. Trace 追踪与可观测性

**Trace 设计**（`docs/TraceLogging设计文档.md`）：

**双轨日志**：
- **JSONL 日志**：结构化追踪，便于程序解析
- **HTML 日志**：人类可读，便于审阅

**追踪内容**：
- 每轮对话的完整 ReAct 过程（Thought / Action / Observation）
- 工具调用的参数、返回值、耗时
- Token 使用统计
- 压缩与 Summary 生成记录
- 错误与异常信息

**脱敏机制**：
- 可配置脱敏开关（`TRACE_SANITIZE=true`）
- 自动隐藏 API Key、密码等敏感信息

**配置项**：
- `TRACE_ENABLED`（默认 true）
- `TRACE_DIR`（默认 memory/traces）
- `TRACE_SANITIZE`（默认 true）
- `TRACE_HTML_INCLUDE_RAW_RESPONSE`（默认 false）

**参考文件**：
- `docs/TraceLogging设计文档.md` - 完整设计文档

## 设计亮点

### 1. 协议优先
- 统一工具响应协议确保所有工具输出格式一致
- 状态机语义清晰（success/partial/error）
- 标准错误码便于错误处理与调试

### 2. 可观测性
- Trace 双轨日志（JSONL + HTML）
- 详细的工具调用统计（耗时、行数、字节数）
- Token 使用追踪与预算管理

### 3. 可扩展性
- MCP 扩展机制支持外部工具接入
- Skill 系统支持自定义技能
- 子代理类型可扩展（general/explore/plan/summary）

### 4. 安全性
- 沙箱隔离（项目根目录限制）
- 工具权限隔离（子代理 deny list）
- 熔断机制（连续失败自动禁用）
- 敏感信息脱敏

### 5. 用户体验
- Enhanced CLI UI（工具调用树、进度显示）
- 会话持久化（/save /load）
- @file 强制读取提醒
- 文件新鲜度追踪

---

## 已知限制（MVP）

1. **AgentTeams**：终端交互暂不支持完整 teammate 焦点切换 UI
2. **子代理**：不支持后台执行与 TaskOutput 轮询
3. **@file**：仅支持英文路径，不支持中文、空格、特殊字符
4. **MCP**：需要配置有效的 API Key 才能使用网络工具

